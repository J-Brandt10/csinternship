What was fixed

FIX 1 — Text wrapping
Changed overflow-wrap: anywhere → overflow-wrap: break-word on .feed-group-msg and .feed-msg. The anywhere value broke words mid-character even when a space was nearby, making short words ragged. break-word only breaks genuinely unbreakable tokens (long model numbers, URLs).

FIX 2 — Scrolling
Added .feed-body > * { flex-shrink: 0; }. Without it, flex squishes child rows to ~18 px and clips their text instead of overflowing. min-height: 0 on the body itself lets the container shrink so overflow-y: auto actually engages.

FIX 3 — Error clearing
Added _entryNeedsResponse() which is false for bare errors (unlike _entryPending). The ✕ Clear button and per-entry dismiss ✕ now use this predicate, so red error notifications can finally be removed. A 30-second auto-sweep clears resolved entries automatically.

Scroll the feed on the left to see Fix 2. Expand the red group to see per-entry dismiss ✕ (Fix 3). Click the header ✕ to clear all clearable entries.
